# 301 重定向 今天遇到一个 301 重定向问题，访问链接走的是 https 没有问题，但是会被重定向到 http 协议，导致文章的归档中，跳转有问题抛出异常。 检测发现问题可能出在 CDN 处，当请求被送到 cdn 处后，cdn 与服务器（此处为 github Page）链接不太正常，只能走 http。 当关闭 CDN，直连 Github 没啥问题了。 但问题是，不用 CDN 国内访问 GitHub 速度感人，所以 CDN 得用。 修改脚本文件改个路径，不重定向，关于原理，搜一下 https 请求原理去学习一下吧。
